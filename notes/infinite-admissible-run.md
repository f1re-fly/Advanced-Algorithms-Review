# 无限 admissible 非决定执行

## 目标

Lemma 2 给出初始二价配置，Lemma 3 保证任意当前可执行事件都能作为某段有限 schedule 的最后一步执行，并让末尾仍为二价。最后一步要把这些有限 schedule 接成无限 run，同时证明：

1. 至多一个进程只执行有限步；
2. 发给持续运行进程的每条消息最终被接收；
3. 没有配置进入决定状态。

## 分阶段构造

维护一个包含全部进程的轮转队列，初始次序任意。消息缓冲区中的消息按发送时间排序，较早发送的排在前面；时间相同的消息使用任意但固定的次序。

假设一轮开始于二价配置 `C`：

1. 取队首进程 `p`。
2. 查看本轮开始时的缓冲区。若存在发给 `p` 的消息，取最早的 `m`；否则使用空接收。记事件 `e = (p, m)`。
3. 由 Lemma 3，存在一段从 `C` 出发的有限 schedule，其最后一个事件是 `e`，且终点 `C'` 仍是二价配置。
4. 把这段 schedule 作为本轮，并把 `p` 移到进程队尾。

每轮至少执行目标事件 `e`，因此无限多个轮次确实产生一条无限 run，而不是无限等待在某个有限前缀上。

把调度器写成伪代码，循环里真正需要选择的只有 Lemma 3 返回的有限 schedule：

```text
C ← 一个初始二价配置
Q ← 所有进程的一个固定排列

forever:
    p ← Q 的队首
    m ← 本轮开始时发给 p 的最早待收消息；若没有则为空
    σ ← 一段以 (p, m) 收尾、终点仍二价的有限 schedule
    执行 σ，并把 p 移到 Q 的队尾
```

## 为什么每个进程都执行无限多步

进程数有限，队列每轮把队首移动到队尾。任意进程 `p` 在至多一个完整轮转后都会再次成为队首。以 `p` 为队首的轮次最终执行事件 `(p, m)`，所以 `p` 至少执行一步。

无限多个轮转中，每个进程都无限多次成为队首，也就执行无限多个步骤。因此构造中没有 faulty process，当然满足“至多一个 faulty process”。

## 为什么每条消息最终被接收

取任意一条发给进程 `p` 的消息 `x`。`x` 发出时，整个 run 只经过了有限多个步骤；每步只发送有限条消息，所以排在 `x` 前面的、发给 `p` 的更早消息只有有限条。

此后每当 `p` 成为队首，本轮都会保证接收轮次开始时最早的一条待收消息。Lemma 3 生成的轮内 schedule 还可能额外接收其他消息，但不会让进度倒退；在 `x` 之后发出的消息也不会排到 `x` 前面。经过有限多次 `p` 的轮次后，所有比 `x` 更早的待收消息都已移除，`x` 随即成为目标事件并被接收。因此，发给每个持续运行进程的每条消息最终都会被接收。

也可以把排在 `x` 前面的待收消息数看成一个非负整数。每逢 `p` 轮到队首，只要 `x` 尚未接收，这个数就至少减一；它不可能无限下降。等它变成零，下一次 `p` 到队首时就会选中 `x`。这个排名论证排除了“每次都送一条消息，但某条老消息仍被永久拖住”的漏洞。

## 为什么整条 run 都不决定

构造保证每轮终点二价。轮内的中间配置也必须二价：如果某个中间配置已经单价，它的任何后继都只能保持同一个价，不可能在本轮终点恢复为二价。

已经包含决定值的可达配置必然单价。进程决定后不能撤回输出，而 partial correctness 的 agreement 排除了未来出现另一个决定值。因此，二价配置中没有进程已经决定。

## 合并结论

该 run 是无限的；每个进程执行无限多步；每条发给进程的消息最终接收；同时没有进程决定。它因此是一条 admissible nondeciding run，与“每条 admissible run 最终决定”的 total correctness 假设直接矛盾。
